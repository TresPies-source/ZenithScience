# ZenithScience Portal v1.2: The Living Response

**Author:** Alfonso Morales (Dojo Genesis)
**Status:** Draft
**Created:** 2026-02-20
**Grounded In:** v1.1 (MCP infrastructure fix, PDF pipeline, patch_intent prompt hardening)

---

## 1. Vision

> Replace the frozen-screen wait with a living, breathing response: tokens flow as they arrive, tool calls appear in real-time, and patch intents surface the instant the agent commits to an edit.

**The Core Insight:**
The portal's agent chat currently blocks for 3-15 seconds while the LLM thinks, executes tools, and composes its response. The user stares at bouncing dots. This latency is structural, not accidental — Ollama on a local M-series chip needs those seconds. But perceived latency can drop to near-zero if the first token appears within 200ms of the LLM emitting it.

The gateway already has a mature SSE event system (36 event types across 3 generations: v0.0.17 core, v0.0.18 artifacts, v0.0.30 orchestration) and a working OpenAI-compatible streaming path. The agent chat endpoint is the only major route that doesn't stream. v1.2 closes this gap by wiring the existing `StreamingAgent` pattern into the gateway agent chat handler and building a frontend event consumer in the portal.

**What Makes This Different:**
This is not a ground-up streaming implementation. The event types, chunking logic, SSE wire format, and even the `StreamingAgentWithEvents` class already exist. v1.2 is a **wiring release** — connecting existing backend infrastructure to a new frontend consumer. The risk is low, the payoff is immediate, and the architecture doesn't change. We're turning on a light that was already installed.

---

## 2. Goals & Success Criteria

**Primary Goals:**
1. Stream agent chat responses token-by-token from gateway to portal UI
2. Show real-time tool invocation events (tool_invoked → tool_completed lifecycle)
3. Surface patch_intent proposals as soon as they're parsed, not after the full response

**Success Criteria:**
- First response token visible in UI within 500ms of LLM emitting it
- Tool call indicators appear immediately when LLM requests a tool (not after response completes)
- Non-streaming fallback works identically to v1.1 (backward compatible)
- `cargo test` and `go test` both pass with streaming enabled
- No regression in patch_intent detection or document context injection

**Non-Goals (Out of Scope):**
- Real-time collaborative editing (multi-user streaming)
- WebSocket transport (SSE is sufficient for unidirectional server→client)
- Streaming the orchestration DAG execution (v1.3 candidate)
- Provider-level token streaming from Ollama (already works via `GenerateCompletionStream`)

```
│      ├─ emit thinking                                       │

│      │    ├─ emit tool_completed                            │
│      │    └─ loop (up to maxIter=8)                         │
│      ├─ final response → splitIntoChunks(50 words)          │
│      │    └─ emit response_chunk (× N)                      │
│      ├─ parsePatchIntent() → emit patch_intent              │
│      └─ emit complete (usage stats)                         │
└─────────────────────────────────────────────────────────────┘
```

**Key Design Decision: Bypass Tauri IPC for streaming.**

Tauri's `invoke()` is request-response. Streaming requires either Tauri events (`app.emit()`) or direct HTTP from the frontend. Direct HTTP via `fetch()` + `ReadableStream` is simpler, doesn't require Rust middleware changes, and lets the Svelte frontend talk directly to the gateway's SSE endpoint. The gateway already runs on `localhost:8080` and the portal knows its address from `AppConfig.gateway_host` / `gateway_port`.

The non-streaming path (`stream` absent or `false`) continues through Tauri IPC unchanged — full backward compatibility.

```go
// handleGatewayAgentChatStream handles the SSE streaming variant of agent chat.
// POST /v1/gateway/agents/:id/chat with {"stream": true}
func (s *Server) handleGatewayAgentChatStream(c *gin.Context, agentID string, req *agentChatRequest, agentConfig *gateway.AgentConfig) {
    eventChan := make(chan events.StreamEvent, 100)

    // SSE headers
    c.Header("Content-Type", "text/event-stream")
    c.Header("Cache-Control", "no-cache")
    c.Header("Connection", "keep-alive")
    c.Header("X-Accel-Buffering", "no")

    flusher, ok := c.Writer.(http.Flusher)
    if !ok {
        s.errorResponse(c, http.StatusInternalServerError, "server_error", "Streaming not supported")
        return
    }

    llmProvider, err := s.resolveProvider("")
    if err != nil {
        s.writeSSEEvent(c.Writer, flusher, events.NewErrorEvent(err.Error(), "PROVIDER_ERROR"))
        return
    }

    systemPrompt := s.buildAgentSystemPrompt(agentConfig, req.DocumentID, req.DocumentContent)
    messages := []provider.Message{
        {Role: "system", Content: systemPrompt},
        {Role: "user", Content: req.Message},
    }
    toolList := s.buildToolList(c.Request.Context())

    llmCtx, cancel := context.WithTimeout(context.Background(), 5*time.Minute)
    defer cancel()

    // Launch the streaming agent loop in a goroutine
    go s.runAgentLoopStreaming(llmCtx, agentID, llmProvider, messages, toolList, eventChan)

    // Drain the event channel into SSE
    for evt := range eventChan {
        s.writeSSEEvent(c.Writer, flusher, evt)
    }

    // Terminator
    fmt.Fprintf(c.Writer, "data: [DONE]\n\n")
    flusher.Flush()
}
```

**New function — `runAgentLoopStreaming`:**

```go
// runAgentLoopStreaming drives the LLM ↔ tool loop while emitting events.
func (s *Server) runAgentLoopStreaming(
    ctx context.Context,
    agentID string,
    llmProvider provider.ModelProvider,
    messages []provider.Message,
    tools []provider.Tool,
    eventChan chan<- events.StreamEvent,
) {
    defer close(eventChan)
    startTime := time.Now()
    const maxIter = 8

    // Thinking indicator
    eventChan <- events.NewThinkingEvent("Processing your request...")

    var finalResp *provider.CompletionResponse
    var lastErr error

    for i := 0; i < maxIter; i++ {
        resp, err := llmProvider.GenerateCompletion(ctx, &provider.CompletionRequest{
            Messages: messages,
            Tools:    tools,
        })
        if err != nil {
            eventChan <- events.NewErrorEvent(err.Error(), "LLM_ERROR")
            return
        }

        if len(resp.ToolCalls) == 0 {
            finalResp = resp
            break
        }

        // Emit tool events
        assistantMsg := provider.Message{Role: "assistant", Content: resp.Content, ToolCalls: resp.ToolCalls}
        messages = append(messages, assistantMsg)

        for _, tc := range resp.ToolCalls {
            eventChan <- events.NewToolInvokedEvent(tc.Name, tc.Arguments)
            tcStart := time.Now()
            result := s.executeToolCall(ctx, tc)
            durationMs := time.Since(tcStart).Milliseconds()
            eventChan <- events.NewToolCompletedEvent(tc.Name, result, durationMs)

            messages = append(messages, provider.Message{
                Role: "tool", Content: result, ToolCallID: tc.ID,
            })
        }
    }

    if finalResp == nil {
        // maxIter reached — one final completion without tools
        resp, err := llmProvider.GenerateCompletion(ctx, &provider.CompletionRequest{Messages: messages})
        if err != nil {
            eventChan <- events.NewErrorEvent(err.Error(), "LLM_ERROR")
            return
        }
        finalResp = resp
        _ = lastErr
    }

    rawText := finalResp.Content
    if rawText == "" {
        rawText = "I received your message but couldn't generate a response."
    }

    // Stream response in chunks
    patchIntent := parsePatchIntent(rawText)
    cleanText := stripPatchIntent(rawText)

    chunks := splitIntoChunks(cleanText, 50)
    for _, chunk := range chunks {
        eventChan <- events.NewResponseChunkEvent(chunk)
        time.Sleep(50 * time.Millisecond)
    }

    // Emit patch_intent as a dedicated event (if present)
    if patchIntent != nil {
        eventChan <- events.StreamEvent{
            Type: "patch_intent",
            Data: map[string]interface{}{
                "operation":   patchIntent.Operation,
                "section_id":  patchIntent.SectionID,
                "content":     patchIntent.Content,
                "description": patchIntent.Description,
            },
            Timestamp: time.Now(),
        }
    }

    // Complete event with usage
    elapsed := time.Since(startTime).Milliseconds()
    eventChan <- events.NewCompleteEvent(map[string]interface{}{
        "agent_id":      agentID,
        "input_tokens":  finalResp.Usage.InputTokens,
        "output_tokens": finalResp.Usage.OutputTokens,
        "total_tokens":  finalResp.Usage.TotalTokens,
        "duration_ms":   elapsed,
    })
}
```

### 3.5 Frontend: Streaming Fetch Client

**Purpose:** Consume SSE events from the gateway using `fetch()` + `ReadableStream`, bypassing Tauri IPC for the streaming path only.

**File:** `src/lib/api/gateway-stream.ts` (new file)

```typescript
import type { PatchIntent } from '$lib/types/index.js';

// ─── Event Types ──────────────────────────────────────────────────────────────

export type StreamEventType =
    | 'thinking'
    | 'tool_invoked'
    | 'tool_completed'
    | 'response_chunk'
    | 'patch_intent'
    | 'complete'
    | 'error';

export interface StreamEvent {
    type: StreamEventType;
    data: Record<string, unknown>;
    timestamp: string;
}

export interface StreamCallbacks {
    onThinking?: (message: string) => void;
    onToolInvoked?: (tool: string, args: Record<string, unknown>) => void;
    onToolCompleted?: (tool: string, result: unknown, durationMs: number) => void;
    onChunk?: (content: string) => void;
    onPatchIntent?: (patch: PatchIntent) => void;
    onComplete?: (usage: Record<string, unknown>) => void;
    onError?: (message: string, code: string) => void;
}

// ─── Streaming Chat ───────────────────────────────────────────────────────────

/**
 * Send a chat message to the gateway agent with streaming enabled.
 * Parses SSE events from the response and dispatches them to callbacks.
 *
 * Falls back to null if the gateway doesn't support streaming (HTTP != 200
 * or Content-Type != text/event-stream), allowing the caller to fall back
 * to the non-streaming Tauri IPC path.
 */
export async function chatWithAgentStream(
    gatewayUrl: string,
    agentId: string,
    message: string,
    callbacks: StreamCallbacks,
    options?: {
        documentId?: string;
        documentContent?: string;
        signal?: AbortSignal;
    }
): Promise<boolean> {
    const url = `${gatewayUrl}/v1/gateway/agents/${agentId}/chat`;
    const body: Record<string, unknown> = {
        message,
        stream: true,
    };
    if (options?.documentId) body.document_id = options.documentId;
    if (options?.documentContent) body.document_content = options.documentContent;

    let response: Response;
    try {
        response = await fetch(url, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(body),
            signal: options?.signal,
        });
    } catch {
        return false; // Network error — caller should fall back
    }

    if (!response.ok || !response.headers.get('content-type')?.includes('text/event-stream')) {
        return false; // Not streaming — caller should fall back
    }

    const reader = response.body?.getReader();
    if (!reader) return false;

    const decoder = new TextDecoder();
    let buffer = '';

    try {
        while (true) {
            const { done, value } = await reader.read();
            if (done) break;

            buffer += decoder.decode(value, { stream: true });
            const lines = buffer.split('\n');
            buffer = lines.pop() ?? '';

            let currentEvent = '';
            let currentData = '';

            for (const line of lines) {
                if (line.startsWith('event: ')) {
                    currentEvent = line.slice(7).trim();
                } else if (line.startsWith('data: ')) {
                    currentData = line.slice(6);
                    if (currentData === '[DONE]') break;

                    try {
                        const evt: StreamEvent = JSON.parse(currentData);
                        dispatchEvent(evt, callbacks);
                    } catch {
                        // Malformed JSON — skip
                    }
                    currentEvent = '';
                    currentData = '';
                } else if (line === '') {
                    // Empty line = event boundary (already handled above)
                    currentEvent = '';
                    currentData = '';
                }
            }
        }
    } finally {
        reader.releaseLock();
    }

    return true;
}

function dispatchEvent(evt: StreamEvent, cb: StreamCallbacks): void {
    switch (evt.type) {
        case 'thinking':
            cb.onThinking?.(evt.data.message as string);
            break;
        case 'tool_invoked':
            cb.onToolInvoked?.(
                evt.data.tool as string,
                evt.data.arguments as Record<string, unknown>
            );
            break;
        case 'tool_completed':
            cb.onToolCompleted?.(
                evt.data.tool as string,
                evt.data.result,
                evt.data.duration_ms as number
            );
            break;
        case 'response_chunk':
            cb.onChunk?.(evt.data.content as string);
            break;
        case 'patch_intent':
            cb.onPatchIntent?.({
                operation: evt.data.operation as string,
                section_id: evt.data.section_id as string | undefined,
                content: evt.data.content as string,
                description: evt.data.description as string,
            });
            break;
        case 'complete':
            cb.onComplete?.(evt.data);
            break;
        case 'error':
            cb.onError?.(evt.data.error as string, evt.data.error_code as string);
            break;
    }
}
```

```typescript
import { chatWithAgentStream, type StreamCallbacks } from '$lib/api/gateway-stream.js';

// New state for streaming
let streamingContent = $state('');
let activeTools = $state<{ tool: string; status: 'running' | 'done'; durationMs?: number }[]>([]);
let isStreaming = $state(false);

async function sendMessage() {
    const agent = $agentStore.agent;
    if (!inputValue.trim() || !agent || sending) return;

    const userMessage = inputValue.trim();
    inputValue = '';
    agentStore.addMessage({ role: 'user', content: userMessage, timestamp: Date.now() });
    sending = true;
    streamingContent = '';
    activeTools = [];

    // Try streaming first
    const gatewayUrl = `http://localhost:${gatewayPort}`;
    const abortController = new AbortController();

    const callbacks: StreamCallbacks = {
        onThinking: () => {
            isStreaming = true;
        },
        onToolInvoked: (tool) => {
            activeTools = [...activeTools, { tool, status: 'running' }];
        },
        onToolCompleted: (tool, _result, durationMs) => {
            activeTools = activeTools.map((t) =>
                t.tool === tool ? { ...t, status: 'done', durationMs } : t
            );
        },
        onChunk: (content) => {
            streamingContent += (streamingContent ? ' ' : '') + content;
        },
        onPatchIntent: (patch) => {
            pendingPatch = patch;
        },
        onComplete: (usage) => {
            // Finalize the message
            agentStore.addMessage({
                role: 'assistant',
                content: streamingContent,
                timestamp: Date.now(),
                toolCalls: activeTools.filter((t) => t.status === 'done').map((t) => ({
                    tool: t.tool, args: {}, result: 'ok'
                })),
            });
            streamingContent = '';
            activeTools = [];
            isStreaming = false;
            sending = false;
            setTimeout(() => chatEndRef?.scrollIntoView({ behavior: 'smooth' }), 50);
        },
        onError: (message) => {
            agentStore.addMessage({
                role: 'assistant',
                content: `Error: ${message}`,
                timestamp: Date.now(),
            });
            isStreaming = false;
            sending = false;
        },
    };

    const streamed = await chatWithAgentStream(
        gatewayUrl, agent.id, userMessage, callbacks,
        { documentId, documentContent, signal: abortController.signal }
    );

    if (!streamed) {
        // Fallback to non-streaming Tauri IPC (identical to v1.1)
        isStreaming = false;
        try {
            const result = await chatWithAgent(agent.id, userMessage, documentId, documentContent);
            // ... existing non-streaming handling (unchanged) ...
        } catch (err) {
            // ... existing error handling (unchanged) ...
        } finally {
            sending = false;
        }
    }
}
```

## 5. Risk Assessment & Mitigation

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Tauri CSP blocks fetch to localhost | Medium | High | Pre-configure CSP in tauri.conf.json; test before UI work |
| Ollama doesn't return tool_calls in streaming mode | Low | Medium | We use `GenerateCompletion` (non-streaming to LLM) inside the loop; only the *response delivery* streams — tool invocation is synchronous |
| SSE connection drops on slow Ollama responses | Medium | Low | 5-minute context timeout; frontend AbortController; automatic reconnect not needed (single-request stream) |
| CORS blocks direct frontend→gateway fetch | Medium | High | Gateway already has AllowedOrigins config; add `tauri://localhost` and `http://localhost:1420` (Vite dev) |
| patch_intent split across chunks | Low | Low | patch_intent is parsed from the complete response *before* chunking — chunks only contain clean text |

## 6. Rollback & Contingency

**Feature Toggle:**
The streaming behavior is gated by `"stream": true` in the request body. If streaming causes issues:
1. Frontend: change `chatWithAgentStream` call to always return `false` → falls back to Tauri IPC
2. Backend: remove the `if req.Stream` branch → all requests go through synchronous path

No database migrations. No schema changes. No protocol changes. Pure additive.

**Rollback Procedure:**
1. Revert `AgentChat.svelte` to v1.1 (remove streaming imports and state)
2. Revert `handle_gateway.go` to v1.1 (remove stream branch)
3. Rebuild gateway binary, copy to portal
4. `cargo build` — done

**Monitoring:**
- Gateway logs: watch for `LLM_ERROR` events in streaming path
- Frontend: `console.warn` on streaming fallback (indicates gateway doesn't support SSE)

## 7. Wire Protocol Reference

### SSE Event Stream Format

Each event follows the Server-Sent Events specification:

```
event: {event_type}
data: {"type":"{event_type}","data":{...},"timestamp":"2026-02-20T..."}

```

(Two trailing newlines per event.)

### Event Sequence (typical flow)

```
event: thinking
data: {"type":"thinking","data":{"message":"Processing your request..."},"timestamp":"..."}

event: tool_invoked
data: {"type":"tool_invoked","data":{"tool":"zen_sci_latex:convert_to_pdf","arguments":{"source":"..."}},"timestamp":"..."}

event: tool_completed
data: {"type":"tool_completed","data":{"tool":"zen_sci_latex:convert_to_pdf","result":"...","duration_ms":1850},"timestamp":"..."}

event: response_chunk
data: {"type":"response_chunk","data":{"content":"I've converted your document to PDF."},"timestamp":"..."}

event: response_chunk
data: {"type":"response_chunk","data":{"content":"The resulting file is 2 pages"},"timestamp":"..."}

event: response_chunk
data: {"type":"response_chunk","data":{"content":"with standard LaTeX formatting."},"timestamp":"..."}

event: patch_intent
data: {"type":"patch_intent","data":{"operation":"replace","section_id":"intro-1","content":"...","description":"Fix formatting"},"timestamp":"..."}

event: complete
data: {"type":"complete","data":{"agent_id":"abc-123","input_tokens":150,"output_tokens":320,"total_tokens":470,"duration_ms":3250},"timestamp":"..."}

data: [DONE]
```

### Error Event

```
event: error
data: {"type":"error","data":{"error":"LLM provider timeout","error_code":"LLM_ERROR"},"timestamp":"..."}

data: [DONE]
```

## 9. Future Considerations

**v1.3 Candidates:**
- **Provider-level token streaming**: Use `GenerateCompletionStream` inside the agentic loop so tokens arrive as the LLM generates them (currently we wait for full completion per iteration, then chunk the response)
- **Orchestration DAG streaming**: Stream `orchestration_*` events through the same SSE path
- **Streaming cancellation**: Frontend sends abort → gateway cancels LLM context mid-generation
- **Multi-turn streaming**: Stream tool call results back to the user as they complete (progressive disclosure)

**v1.4 Candidates:**
- **WebSocket upgrade**: If bidirectional communication becomes necessary (e.g., user interrupts, changes context mid-stream)
- **Server-Sent Events via Tauri**: Use `app.emit()` / `listen()` instead of direct HTTP, removing the CSP requirement
